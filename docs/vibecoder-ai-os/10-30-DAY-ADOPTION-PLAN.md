# 30-Day Adoption Plan

## Day 1 — Stabilize

- Revoke exposed keys.
- Back up `.codex`.
- Confirm native and gateway launch.
- Keep `lean` working.
- Do not add new providers yet.

## Days 2–3 — Active pool

Test:

- Terra,
- Luna,
- Sol,
- Gemini candidate,
- GLM,
- DeepSeek,
- MiniMax.

Promote only proven routes.

## Days 4–5 — Context layer

Create:

- global AGENTS,
- repository AGENTS,
- architecture,
- current status,
- release checklist,
- checkpoint.

Use a compatible native/Terra or MCP-tested Gemini session to make these repository-specific.

## Week 2 — Skills and reviewers

Install:

- feature-build,
- root-cause-debug,
- release-verification,
- UI polish,
- database-safe-change.

Add read-only custom agents. Set max threads to 3 and depth to 1.

## Week 3 — Browser/mobile quality

- formalize browser verification,
- add two-user challenge testing,
- verify PWA refresh/update,
- establish Android device checklist,
- remove duplicated or low-value tests.

## Week 4 — Optimize routing

Review your actual logs:

- which model completed tasks,
- which failed tools,
- which hallucinated,
- which was fast,
- where Sol changed the decision,
- where external models were sufficient.

Demote weak routes. Add one optional provider only if it solves a real recurring shortage.

## Final target

By day 30:

- no repeated repository rediscovery,
- one clear command for each task class,
- native usage reserved for high leverage,
- external models verified rather than assumed,
- every meaningful task ends with evidence and checkpoint,
- releases have an honest readiness verdict.
