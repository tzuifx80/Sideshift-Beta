---
name: ui-polish
description: Use for a concrete UI quality, responsiveness, accessibility, or visual-consistency task; not for vague redesign brainstorming.
---

1. Define the user-visible problem and target viewports.
2. Inspect design tokens and shared components.
3. Capture current states: normal, loading, empty, error, disabled, and long content.
4. Fix hierarchy, spacing, responsiveness, and accessibility using existing patterns.
5. Avoid one-off styling and unrelated redesign.
6. Test keyboard focus, contrast, motion, and overflow.
7. Run component/browser checks.
8. Inspect visual diff and report manual viewport checks.
