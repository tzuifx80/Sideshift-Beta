---
name: root-cause-debug
description: Diagnose a bug by tracing causality before editing.
---
1. Define symptom and expected result.
2. Reproduce or collect strongest evidence.
3. Trace callers, state transitions, persistence, async boundaries, adapters, and errors.
4. Rank and falsify hypotheses.
5. Identify root cause before editing.
6. Implement smallest complete fix and regression test.
7. Run affected checks and inspect final diff.
8. Report root cause separately from patch.
