name: design-quality
description: Use for UI, responsive design, accessibility, visual hierarchy, or interaction polish.
---
1. Define user problem, screens, states, and target viewports.
2. Inspect design tokens and shared components.
3. Capture current behavior.
4. Improve hierarchy, consistency, responsive behavior, semantics, focus, contrast, and reduced motion.
5. Verify real content plus loading, empty, error, offline, disabled, and unavailable states.
6. Run browser/visual checks and inspect final diff.
